UI Mihomo / Ultra — обновление переноса v1.2.101

Что изменилось в этом релизе:
- релиз `v1.2.101` продолжает перевод Mihomo-конфига в читаемые формы: переработаны экраны `proxy-providers` и `rule-providers`, добавлены поиск по списку, quick-presets и более явные смысловые блоки формы
- `proxy-providers` теперь читается как карточка сущности: профиль типа (`http / file / inline`), отдельные блоки для источника, фильтрации, health-check и advanced YAML, плюс быстрые пресеты для health-check URL
- `rule-providers` тоже стал понятнее: профиль поведения (`classical / domain / ipcidr`), поиск по списку и разбиение формы на блоки `идентификация / поведение`, `источник / обновление` и `advanced YAML`
- `router-agent` в этом релизе не менялся; линия остаётся `0.6.22`

Текущие версии:
- UI: v1.2.101
- v1.2.101: form-driven шаг для `proxy-providers / rule-providers`; добавлены поиск по списку, type/behavior-профили, quick-presets и более понятное разбиение формы по смысловым блокам; router-agent не менялся
- router-agent: 0.6.22

Что важно по интерфейсу:
- `Mihomo` остаётся отдельным рабочим разделом
- `Settings` держит только интерфейс, поведение и связанные настройки отображения
- safe managed-config flow сохранён: `draft / validate / apply / rollback / baseline / history`
- raw YAML-редактор не убирался: structured-блоки работают поверх текущего черновика и не ломают безопасный pipeline
- вкладка `Роутер` по-прежнему разбита на `Обзор / Трафик / Сеть`

Следующий логичный шаг:
- после `v1.2.101` логично добивать ещё более явные сценарии создания сущностей из шаблонов/мастеров, а затем продолжать редкие nested-поля для providers и других секций, чтобы raw YAML всё реже был основным путём
