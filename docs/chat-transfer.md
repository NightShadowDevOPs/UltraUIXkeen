UI Mihomo / Ultra — обновление переноса v1.2.105

Что изменилось в этом релизе:
- релиз `v1.2.105` делает мастер создания `proxies` более сценарным: теперь есть не только общие типы, но и готовые варианты `VLESS Reality`, `VLESS WS + TLS`, `VMess WS + TLS`, `Trojan gRPC`, `Trojan TLS`, `WireGuard peer`, `Hysteria2`, `TUIC`
- на шаге `Основное` мастер показывает summary по выбранному сценарию и не даёт перейти к `Проверке`, пока не заполнены ключевые поля для этого сценария
- добавлены отсутствовавшие i18n-строки для мастер/шаблонов, чтобы интерфейс мастеров не показывал сырые ключи перевода
- `router-agent` в этом релизе не менялся; линия остаётся `0.6.22`

Текущие версии:
- UI: v1.2.105
- v1.2.105: мастер создания `proxies` стал сценарным и начал проверять обязательные поля до шага «Проверка»; router-agent не менялся
- router-agent: 0.6.22

Что важно по интерфейсу:
- `Mihomo` остаётся отдельным рабочим разделом
- `Settings` держит только интерфейс, поведение и связанные настройки отображения
- safe managed-config flow сохранён: `draft / validate / apply / rollback / baseline / history`
- raw YAML-редактор не убирался: structured-блоки работают поверх текущего черновика и не ломают безопасный pipeline
- вкладка `Роутер` по-прежнему разбита на `Обзор / Трафик / Сеть`

Следующий логичный шаг:
- после `v1.2.105` логично идти либо в ещё более умные сценарии мастеров (например, с разной логикой под `provider-backed` vs `manual` группы), либо добивать редкие nested-поля `proxies`, чтобы extra YAML ещё реже был нужен
