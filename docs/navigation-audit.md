# Navigation audit — v1.2.76

## Что изменили

Первый этап разгрузки интерфейса:

- отдельный верхнеуровневый раздел `Mihomo`
- config editor убран из `Settings`
- sidebar разбит на логические секции
- `Settings` перегруппирован в более понятные подблоки

## Текущая структура

### Monitor
- Overview
- Router
- Traffic
- Connections
- Logs

### Network & Mihomo
- Proxies
- Proxy providers
- Subscriptions
- Rules
- Mihomo

### Management
- Tasks
- Users
- Policies
- Settings

## Почему так

До этого `Settings` начинал разрастаться в общий склад всего подряд.
Управление `config.yaml` — это уже не обычная настройка интерфейса, а отдельный операционный контур с draft/baseline/history/rollback, поэтому он вынесен в самостоятельный раздел.

## Что ещё не добито

1. Внутри `Mihomo` пока нет полноценных вложенных подразделов/роутов.
2. Нижний mobile dock всё ещё опирается на общий список route-ов и может потребовать отдельного упрощённого режима.
3. Нужно решить, выносить ли часть `Tasks` в более предметные разделы, если cockpit users DB и сервисные операции продолжат расти.
