# Navigation audit — v1.2.78

## Что изменили

Первый и второй этап разгрузки интерфейса:

- отдельный верхнеуровневый раздел `Mihomo`
- config editor убран из `Settings`
- sidebar разбит на логические секции
- `Settings` перегруппирован в более понятные подблоки
- страница `Mihomo` получила внутренние подразделы `Overview / Runtime / Providers / Rules / Config`
- быстрые переходы внутри Mihomo теперь работают и как внутренний mini-nav, что особенно полезно на узких экранах

## Текущая структура

### Monitor
- Overview
- Router
- Traffic
- Connections
- Logs

### Network & Mihomo
- Proxies
- Proxy providers
- Subscriptions
- Rules
- Mihomo

### Management
- Tasks
- Users
- Policies
- Settings

## Почему так

До этого `Settings` начинал разрастаться в общий склад всего подряд.
Управление `config.yaml` — это уже не обычная настройка интерфейса, а отдельный операционный контур с draft/baseline/history/rollback, поэтому он вынесен в самостоятельный раздел.

## Что ещё не добито

1. Внутри `Mihomo` пока нет полноценных вложенных роутов: это секции одной страницы, а не отдельные страницы.
2. Нижний mobile dock всё ещё опирается на общий список route-ов и может потребовать отдельного упрощённого режима.
3. Нужно решить, выносить ли часть `Tasks` в более предметные разделы, если cockpit users DB и сервисные операции продолжат расти.

## Следующий этап

- решить, нужно ли повышать Mihomo-секции до настоящих вложенных роутов
- отдельно сделать аудит мобильной навигации
- при необходимости пересобрать `Tasks` и `Settings` ещё на один уровень чище
